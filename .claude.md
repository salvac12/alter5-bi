# Alter5 Business Intelligence

## Descripción del Proyecto

Herramienta de inteligencia comercial para analizar, clasificar y priorizar la red de contactos empresariales de Alter5.

**Contexto de negocio**: Alter5 se dedica a la financiación y M&A de proyectos y empresas vinculadas a la transición energética (renovables, autoconsumo, movilidad, green data centers). Esta herramienta permite identificar oportunidades comerciales mediante el análisis de las interacciones históricas de múltiples buzones de correo.

**Objetivo principal**: Identificar y priorizar empresas con alto potencial comercial basándose en:
- Volumen de interacciones
- Recencia del contacto
- Amplitud de la red de contactos
- Relevancia estratégica del tipo de relación

## Stack Tecnológico

- **Frontend**: React 18.3.1 + Vite 5.4.0
- **Estilado**: CSS-in-JS inline (sin dependencias de UI)
- **Utilidades**: Lodash 4.17.21
- **Importación de datos**: Python 3.x + pandas + openpyxl
- **Deploy**: Vercel (static site)

## Sistema de Scoring (0-100)

El score total se calcula sumando 4 dimensiones:

| Dimensión | Peso máx | Fórmula | Qué mide |
|-----------|----------|---------|----------|
| **Volumen** | 35 | `min(35, log(interacciones+1) / log(max) * 35)` | Intensidad de la relación (logarítmico) |
| **Recencia** | 30 | `max(0, 30 - mesesDesdeUltimo * 1.5)` | Cuán activa está la relación |
| **Red** | 15 | `min(15, nContactos * 3)` | Amplitud de contactos en la empresa |
| **Tipo** | 20 | `max(weights[tipos])` | Relevancia estratégica |

### Pesos por tipo de relación

Ver `src/utils/constants.js`:
```js
{
  "Potencial Prestatario": 20,  // ⭐ Máxima prioridad
  "Inversor/Fondo": 18,
  "Partnership": 16,
  "Banco": 15,
  "Asesor Financiero": 12,
  // ...
}
```

### Estados de relación

- **Activa** (verde): último contacto < 6 meses
- **Dormida** (ámbar): entre 6 y 18 meses
- **Perdida** (rojo): > 18 meses

**Fecha de referencia**: `2026-02-14` (configurable en `src/utils/constants.js:REF_DATE`)

## Deploy a Vercel

**Documentación completa:** [DEPLOY.md](./DEPLOY.md) — requisitos, script `deploy-vercel.sh`, deploy manual con CLI, `vercel.json` / `.vercelignore`, primer deploy, `vercel --prod --yes`, GitHub, dominio y troubleshooting.

### Resumen rápido

- **Script:** `./deploy-vercel.sh` (verifica Node, deps, build, Git, Vercel CLI y ejecuta `vercel --prod`).
- **Manual:** `npm install` → `npm run build` → `vercel login` (primera vez) → `vercel --prod` (o `vercel --prod --yes` en CI).
- **Configuración:** `vercel.json` en raíz (build, output `dist`, framework Vite, headers). `.vercelignore` excluye `node_modules`, `data_sources`, `scripts`, etc.
- **Proyecto enlazado:** Tras el primer deploy, la CLI crea `.vercel/`; los siguientes deploys usan ese proyecto sin preguntar.

### Variables de entorno (futuro)

Para Airtable (cuando se implemente):
```bash
# En Vercel Dashboard → Settings → Environment Variables
VITE_AIRTABLE_API_KEY=keyXXXXXXXXXXXXXX
VITE_AIRTABLE_BASE_ID=appXXXXXXXXXXXXXX
VITE_AIRTABLE_TABLE_NAME=Empresas
```

### Notas importantes

- Los archivos `src/data/*.json` deben estar en el repo para que el build funcione en Vercel.
- `data_sources/` NO se sube a git (está en `.gitignore`); los datos de la app vienen de `companies.json` generado por los scripts.

## Mejores Prácticas

### Al trabajar con datos

1. **Nunca editar manualmente** `companies.json` o `companies_full.json`
   - Siempre usar `scripts/import_mailbox.py`
   - Los archivos se regeneran automáticamente

2. **Backup de datos antes de re-importar**
   ```bash
   cp src/data/companies_full.json src/data/companies_full.backup.json
   ```

3. **Formato de Excel requerido**
   - Hoja "Resumen por Empresa": empresa, dominio, sector, contactos, interacciones, tipo, fechas
   - Hoja "Clasificacion IA": empresa, contacto, cargo
   - Hoja "Linea Temporal": empresa, trimestre, nº emails

### Al desarrollar nuevos componentes

1. **Mantener consistencia de diseño**
   - Colores: Usar gradiente `linear-gradient(135deg, #3B82F6, #10B981)`
   - Tipografía: DM Sans (weights: 400, 500, 600, 700, 800)
   - Espaciado: Múltiplos de 4px (8, 12, 16, 20, 24...)

2. **Performance**
   - Usar `useMemo` para cálculos costosos (scoring, filtrado)
   - Paginación obligatoria para listas >100 items
   - Lazy load del `DetailPanel` (solo renderiza cuando hay empresa seleccionada)

3. **Estado global**
   - Por ahora todo en `App.jsx` (prop drilling)
   - Si crece: migrar a Context API o Zustand

### Al modificar el scoring

1. **Documentar cambios** en este archivo y en `README.md`
2. **Actualizar `REF_DATE`** si es necesario (`src/utils/constants.js:42`)
3. **Considerar impacto** en empresas existentes (puede cambiar rankings)

## Preguntas Frecuentes

### ¿Cómo añado un nuevo sector?

Editar `src/utils/constants.js`:
```js
export const SECTORS = [
  "Asesor Financiero", "Banca", "Construcción",
  "TU_NUEVO_SECTOR", // ← Aquí
  // ...
];
```

### ¿Cómo cambio los pesos del scoring?

Editar `src/utils/constants.js`:
```js
export const TYPE_WEIGHTS = {
  "Potencial Prestatario": 20, // ← Ajustar aquí
  // ...
};
```

Luego reiniciar el servidor de desarrollo.

### ¿Cómo borro una empresa?

Actualmente no hay UI para eliminar. Para borrar manualmente:
1. Hacer backup de `src/data/companies_full.json`
2. Editar el JSON y eliminar la entrada
3. Re-ejecutar `python scripts/import_mailbox.py` para regenerar `companies.json`

### ¿Puedo importar el mismo buzón dos veces?

Sí, el script actualiza el buzón existente (reemplaza datos del empleado, recalcula agregados).
